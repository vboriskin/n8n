Да — вот уже **карта переподключения руками в n8n**, без воды.

Сразу 2 важных вывода по текущему JSON:

* `Apply Business Rules` сейчас действительно затирает весь вход и возвращает только `competency_analysis`, из-за этого дальше ломается planning и финальная сборка 
* в workflow есть старая LLM-ветка, где prompt ждёт `{{$json.payload}}`, поэтому её лучше выключить, чтобы не путалась с новой логикой 
* в sample input матрицы лежат с полем `compentecy_groups`, а финальный отчёт уже считает `role_matrix.competencies`, что требует нормализации схемы

# 3. Переподключение по шагам

## Шаг 1. Вход

### Связи

```text
Edit Fields5 -> Code in JavaScript12
Code in JavaScript12 -> Build Employee Context1
```

### Что должно быть после `Build Employee Context1`

Один item на сотрудника:

```json
{
  "employee": {...},
  "self_assessment": {...},
  "role_matrix": {...},
  "learning_assets_pool": [...],
  "employee_data_quality_issues": [...],
  "config": {...}

```

### Проверка

Если тут нет `role_matrix.competencies`, дальше можно не идти.

```text
Build Employee Context1 -> Build Competency Items -> Score Competencies
```

### Почему

`Score Competencies` ждёт `competency_items`, а сейчас выше они не формируются.

### Настройка

* тип: `Code`

```text
Shortlist Competencies -> Code in JavaScript3
```

# 4. Как правильно подключить fallback после competency LLM

Сейчас fallback у вас логически нужен, но его надо встроить в поток.

## Добавьте `IF`

Название:
`IF Competency Result Valid?`

### Связи

```text
Parse/Validate Competency Result -> IF Competency Result Valid?
```

### Условие

Boolean condition:

```js
{{$json.valid}} is true
```

---

## Дальше две ветки

### Ветка true

```text
IF Competency Result Valid? [true] -> Merge Competency Result
```

### Ветка false

```text
IF Competency Result Valid? [false] -> Fallback result -> Merge Competency Result
```

---

## Merge для competency results

### Нода

Название:
`Merge Competency Result`

### Тип merge

Если на входе всегда по одному item:

* `Append`

Если хотите склеивать поля:

* лучше не надо на этом этапе
  тут нужен именно проход обоих результатов в общий downstream.

### Дальше

```text
Merge Competency Result -> Split In Batches (next batch)
```

или, если у вас уже стоит collector:

```text
Merge Competency Result -> Collect Results
```

---

# 5. Как собрать competency results назад

Если вы используете batch loop, то схема такая:

```text
Code in JavaScript3
-> Split In Batches
-> Basic LLM Chain1
-> Parse/Validate Competency Result
-> IF
   -> true -> Merge Competency Result
   -> false -> Fallback result -> Merge Competency Result
-> Collect Results
```

### Важно

`Collect Results` должен быть после того места, где уже все competency items либо провалидированы, либо ушли в fallback.

# 9. Как собрать manager + one2one + core context в один item

Здесь нужно 2 merge-ноды.

---

## Merge 1: manager в основной поток

### Нода

`Merge Manager Outputs`

### Вход 1

`Repair Plan Constraints`

### Вход 2

`Parse / Validate Manager Pla`

### Режим

Если один сотрудник за запуск:

* `Combine`
* `Merge By Position`

Если несколько сотрудников:

* лучше `Merge By Fields`, по `employee.employee_id`

---

## Merge 2: one2one в уже собранный item

### Нода

`Merge Final Inputs`

### Вход 1

`Merge Manager Outputs`

### Вход 2

`Parse/Validate One2One Guide`

### Режим

Такой же:

* `Merge By Position` для одного сотрудника
* `Merge By Fields` для нескольких

# 14. Что проверять после каждого этапа

## После `Build Employee Context1`

* есть `employee`
* есть `role_matrix`
* есть `role_matrix.competencies`

## После `Build Competency Items`

* `competency_items.length > 0`

## После `Code in JavaScript3`

* каждый item = одна компетенция

## После `Parse/Validate Competency Result`

* часть ответов `valid=true`

## После `Fallback result`

* на invalid input тоже есть `competency_result`

## После `Collect Results`

* есть массив `competency_results`

## После `Apply Business Rules`

* сохранились `employee`, `role_matrix`, `config`

## После `Merge Final Inputs`

* в одном item одновременно есть:

  * `development_plan`
  * `manager_plan`
  * `one2one_guide`

## После `Build Final Report JSON`

* `employee_report_json` полный
* `role_matrix.competencies_count` не 0, если матрица нормализована

# 15. Главный риск после переподключения

Даже если связи станут правильными, у вас останутся 3 обязательные правки в коде:

1. `Code in JavaScript12`
2. новая `Build Competency Items`
3. `Apply Business Rules` с `...$json`

Без них сама схема будет верная, но данные всё равно не дойдут как надо.

Следующим сообщением я могу дать вам **прямо готовую схему “что в какой input merge-ноды подключать” в виде короткого чеклиста для ручной сборки в редакторе n8n**.
