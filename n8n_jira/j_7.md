# Каркас n8n workflow

## 0. `Set Input`

Режим: обычный

Пример входа:

```json
{
  "employee": {
    "employee_id": "87654321",
    "full_name": "Иванов Иван Иванович",
    "email": "user@delta.sbrf.ru",
    "role_id": "developer",
    "grade_id": "senior"
  },
  "workflow_config": {
    "jira_base_url": "https://jira.delta.sbrf.ru",
    "lookback_days": 180,
    "default_priority_weight": 2,
    "include_competencies_without_self_assessment": true,
    "include_competencies_without_evidence": true
  },
  "competency_catalog": {
    "competencies": []
  },
  "assessment_input": {
    "self_assessments": []
  },
  "scoring_input": {},
  "jira_token": "YOUR_TOKEN"
}
```

## 1. `HTTP Request / Find Jira User By Email`

Method: `GET`

URL:

```text
={{$json.workflow_config.jira_base_url}}/rest/api/2/user/search
```

Query params:

* `username` = `={{$json.employee.email}}`

Headers:

* `Content-Type` = `application/json`
* `Authorization` = `Bearer {{$json.jira_token}}`

## 4. `HTTP Request / Jira Search Issues`

Method: `POST`

URL:

```text
={{$json.workflow_config.jira_base_url}}/rest/api/2/search
```

Headers:

* `Content-Type` = `application/json`
* `Authorization` = `Bearer {{$json.jira_token}}`

Body JSON:

```javascript
={{
{
  jql: `(
    assignee = "${$json.jira_user.key || $json.jira_user.name}"
    OR reporter = "${$json.jira_user.key || $json.jira_user.name}"
    OR worklogAuthor = "${$json.jira_user.key || $json.jira_user.name}"
  ) AND updated >= -${$json.workflow_config.lookback_days || 180}d ORDER BY updated DESC`,
  startAt: 0,
  maxResults: 100,
  fields: [
    "summary",
    "description",
    "labels",
    "components",
    "issuetype",
    "status",
    "resolution",
    "assignee",
    "reporter",
    "created",
    "updated",
    "comment",
    "worklog",
    "timespent",
    "timeoriginalestimate",
    "aggregatetimespent",
    "aggregatetimeoriginalestimate",
    "timetracking"
  ],
  expand: ["changelog"]
}
}}
```

## 9. `Code / Build Dev Status Map`

Mode: `Run Once for All Items`

```javascript
const map = {};

for (const item of items) {
  const issueKey = item.json.issue_key || item.json.base_issue?.key || item.json.key;
  if (!issueKey) continue;

  map[issueKey] = {
    summary: item.json.summary || null,
    detail: item.json.detail || []
  };
}

return [{ json: { dev_status_by_issue: map } }];
```

## 12. `Code / Build Full Jira Input`

Mode: `Run Once for All Items`

Эта нода собирает root input + issue details + optional dev maps.

```javascript
const root = $item(0).$node["Merge / Attach Jira User"].json;
const issueItems = $items("HTTP Request / Jira Issue Details", 0, 0);

let devMap = {};
let changelogMap = {};

try {
  const devNodeItems = $items("Code / Build Dev Status Map", 0, 0);
  if (devNodeItems.length) devMap = devNodeItems[0].json.dev_status_by_issue || {};
} catch (e) {}

try {
  const changelogNodeItems = $items("Code / Build Changelog Dev Map", 0, 0);
  if (changelogNodeItems.length) changelogMap = changelogNodeItems[0].json.changelog_dev_by_issue || {};
} catch (e) {}

const issues = issueItems.map(i => i.json);

return [{
  json: {
    employee: root.employee,
    jira_token: root.jira_token,
    jira_user: root.jira_user,
    workflow_config: root.workflow_config,
    competency_catalog: root.competency_catalog,
    assessment_input: root.assessment_input,
    scoring_input: root.scoring_input,
    issues,
    dev_status_by_issue: devMap,
    changelog_dev_by_issue: changelogMap
  }
}];
```

## 16. `LLM / Analyze One Competency`

### System

```text
Ты анализируешь одну компетенцию инженера.
Оценивай только на основе переданных данных.
Не выдумывай evidence.

Не путай "нет данных" и "низкий уровень".

Верни строго JSON без markdown.
```


# Связи между нодами

```text
Set Input
  -> HTTP Request / Find Jira User By Email
  -> Code / Pick Jira User
  -> Merge / Attach Jira User
  -> HTTP Request / Jira Search Issues
  -> Code / Split Issues
  -> HTTP Request / Jira Issue Details
      -> IF / Is Developer
          -> true -> HTTP Request / Jira Dev Status -> Code / Build Dev Status Map
          -> false
      -> Code / Extract Dev Signals From Changelog -> Code / Build Changelog Dev Map
  -> Code / Build Full Jira Input
      -> Code / Normalize Jira Evidence
          -> Code / Build Competency Payloads
          -> Code / Pick Top Competencies
          -> LLM / Analyze One Competency
          -> Code / Normalize Competency Result
          -> Code / Build Employee Summary Payload
          -> LLM / Build Employee Summary
          -> Merge / Competency Results + Summary
          -> Code / Build WF30 Output

      -> Code / Build Jira Summary Payload
          -> LLM / Jira Summary
```

# Что я бы упростил в первой рабочей версии

Чтобы быстрее завести MVP:

* сначала не делать `Jira Dev Status`
* оставить только:

  * issue details
  * changelog fallback
  * worklog
* `competency_catalog` сначала держать минимальным
* top competencies = 8–12

То есть MVP:

```text
Set Input
-> Find Jira User
-> Search Issues
-> Issue Details
-> Extract Dev Signals From Changelog
-> Build Changelog Dev Map
-> Build Full Jira Input
-> Normalize Jira Evidence
-> Build Competency Payloads
-> Pick Top Competencies
-> LLM Analyze
-> Summary
```

# Практический чек-лист перед запуском

Проверь отдельно:

* `/rest/api/2/user/search?username=email`
* `/rest/api/2/search`
* `/rest/api/2/issue/{key}?expand=changelog`
* `/rest/dev-status/latest/issue/detail?...` если нужен

И еще:

* `worklogAuthor` в JQL может тормозить
* `dev-status` может быть недоступен
* `Bearer` может не работать, если у вас auth через cookie/basic
