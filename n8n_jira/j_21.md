Да. Ниже даю **готовый adapter layer для n8n Code nodes**, чтобы твой workflow начал принимать этот новый JSON и приводить его к единому контракту.

Я опираюсь на фактическую структуру файла: там есть отдельный блок с `employees` и `employee_issues`, отдельный блок с `matrices`, и еще отдельный блок с `learning_assets` 
Также у сотрудника роль `qa_engineer`, а в матрицах есть, например, `manual_qa_engineer`, поэтому нужен role mapping/fallback 

---

# Куда вставить

Перед твоим текущим `Set Input` / `Build Full Jira Input` добавь новую ветку:

```text
Raw Input
-> Code / Detect Input Format
-> IF / Is Matrix Bundle?
   -> true:
      -> Code / Normalize Matrix Bundle
      -> Code / Pick Employee
      -> Code / Resolve Role Matrix
      -> Code / Build Unified Competency Catalog
      -> Code / Build Learning Assets Index
      -> Code / Build Unified Root Contract
      -> дальше в общий workflow
   -> false:
      -> старый legacy flow
```

const input = $json;

const isArrayRoot = Array.isArray(input);
const blocks = isArrayRoot ? input : [input];

const hasEmployeesBlock = blocks.some(b => Array.isArray(b?.employees));

return [{

  }

# 3. `Code / Normalize Matrix Bundle`

Mode: `Run Once for All Items`

Эта нода распаковывает blocks и достает:

* employee_issues
* matrices

const raw = $json.raw_input;

const employeesBlock = blocks.find(x => Array.isArray(x?.employees)) || {};
const matricesBlock = blocks.find(x => Array.isArray(x?.matrices)) || {};
const learningBlock = blocks.find(x => Array.isArray(x?.learning_assets)) || {};

return [{
  json: {
    input_format: $json.input_format,

    employee_issues: employeesBlock.employee_issues || [],
    matrices: matricesBlock.matrices || [],
    learning_assets: learningBlock.learning_assets || []
  }
}];

# 4. `Code / Pick Employee`

Mode: `Run Once for All Items`

Тут можно выбрать:

* по `target_employee_id`
* по `target_login`
* иначе первого

Я советую в raw input прокидывать `target_employee_id`, если workflow запускается на конкретного человека.

```javascript
const employees = $json.employees || [];

const targetEmployeeId =
  $json.target_employee_id ||
  $item(0).$node["Code / Detect Input Format"].json.target_employee_id ||
  null;

const targetLogin =
  $json.target_login ||
  $item(0).$node["Code / Detect Input Format"].json.target_login ||
  null;

let employee = null;

if (targetEmployeeId) {
  employee = employees.find(e => String(e.employee_id) === String(targetEmployeeId));
}

if (!employee && targetLogin) {
  employee = employees.find(e => String(e.login) === String(targetLogin));
}

if (!employee) {
  employee = employees[0] || null;
}

if (!employee) {
  throw new Error('Employee not found in matrix bundle');
}

return [{
  json: {
    ...$json,
    selected_employee: employee
  }
}];
```

# 5. `Code / Resolve Role Matrix`

Mode: `Run Once for All Items`

Здесь делаем:

* точное совпадение по роли
* fallback через role map
* fallback через grade/position эвристику

```javascript
const employee = $json.selected_employee;
const matrices = $json.matrices || [];

const employeeRole = employee?.role || null;
const employeePosition = (employee?.position || '').toLowerCase();

if (!employeeRole) {
  throw new Error('selected_employee.role is empty');
}

const ROLE_MAP = {
  qa_engineer: [
    'qa_engineer',
    'manual_qa_engineer',
    'automation_qa_engineer'
  ],
  analyst: [
    'analyst',
    'system_analyst',
    'business_analyst'
  ],
  developer: [
    'developer',
    'backend_developer',
    'frontend_developer',
    'fullstack_developer'
  ]
};

const exact = matrices.find(m => m.role === employeeRole);

let resolved = exact || null;
let resolution = 'exact';

if (!resolved) {
  const aliases = ROLE_MAP[employeeRole] || [employeeRole];
  resolved = matrices.find(m => aliases.includes(m.role)) || null;
  if (resolved) resolution = 'alias';
}

if (!resolved) {
  if (employeeRole === 'qa_engineer') {
    if (employeePosition.includes('автомат')) {
      resolved = matrices.find(m => m.role === 'automation_qa_engineer') || null;
      resolution = 'position_fallback';
    } else {
      resolved = matrices.find(m => m.role === 'manual_qa_engineer') || null;
      resolution = 'position_fallback';
    }
  }
}

if (!resolved) {
  throw new Error(`Matrix not found for employee role: ${employeeRole}`);
}

return [{
  json: {
    ...$json,
    selected_matrix: resolved,
    matrix_resolution: resolution
  }
}];
```

# 6. `Code / Build Unified Competency Catalog`

Mode: `Run Once for All Items`

Эта нода превращает `compentecy_groups -> competencies -> levels` в плоский catalog.

Сразу:

* нормализуем id
* сохраняем `group_name`
* сохраняем `role_id`
* вытаскиваем `skill_categories`
* помечаем soft/hard по role и group heuristic

```javascript
function slugify(value) {
  return String(value || '')
    .trim()
    .toLowerCase()
    .replace(/[ё]/g, 'е')
    .replace(/[^\p{L}\p{N}]+/gu, '_')
    .replace(/^_+|_+$/g, '');
}

function detectCompetencyType(groupName, roleId) {
  const g = (groupName || '').toLowerCase();
  const r = (roleId || '').toLowerCase();

  if (r === 'soft_skills') return 'soft';

  const softHints = [
    'личная_эффективность',
    'коммуникац',
    'лидер',
    'команд',
    'настав',
    'обратн',
    'изменени'
  ];

  if (softHints.some(h => g.includes(h))) return 'soft';
  return 'hard';
}

const matrix = $json.selected_matrix;
if (!matrix) throw new Error('selected_matrix is missing');

const roleId = matrix.role;
const groups = matrix.compentecy_groups || [];
const skillCategories = matrix.skill_categories || [];

const flattenedSkills = skillCategories.map(cat => ({
  category: cat.category || null,
  skills: (cat.skills || []).map(s => ({
    skill: s.skill || null,
    description: s.description || ''
  }))
}));

const competencies = [];

for (const group of groups) {
  const groupName = group.group_name || 'unknown_group';
  const compType = detectCompetencyType(groupName, roleId);

  for (const comp of (group.competencies || [])) {
    const competencyName = comp.competency_name || 'unknown_competency';
    const competencyId = slugify(competencyName);

    const levels = (comp.levels || [])
      .map(l => ({
        level: Number(l.level),
        description: l.description || ''
      }))
      .sort((a, b) => a.level - b.level);

    competencies.push({
      competency_id: competencyId,
      competency_name: competencyName,
      group_name: groupName,
      role_id: roleId,
      type: compType,
      levels,
      source: 'matrix_bundle_v1',
      signals: {
        jira: [],
        behavioral: [],
        skill_categories: flattenedSkills
          .filter(x => x.category)
          .map(x => x.category)
      }
    });
  }
}

return [{
  json: {
    ...$json,
    unified_competency_catalog: {
      role_id: roleId,
      matrix_resolution: $json.matrix_resolution || 'unknown',
      competencies,
      skill_categories: flattenedSkills
    }
  }
}];
```

# 7. `Code / Build Learning Assets Index`

Mode: `Run Once for All Items`

В файле есть отдельный блок `learning_assets`, и его очень полезно сразу нормализовать для IDP/learning recommendations 

```javascript
function slugify(value) {
  return String(value || '')
    .trim()
    .toLowerCase()
    .replace(/[ё]/g, 'е')
    .replace(/[^\p{L}\p{N}]+/gu, '_')
    .replace(/^_+|_+$/g, '');
}

const assets = $json.learning_assets || [];

const learningCatalog = assets.map(a => ({
  id: a.id || null,
  title: a.title || null,
  original_title: a.original_title || a.title || null,
  competencies: (a.competencies || []).map(c => slugify(c)),
  matched_competencies_count: Number(a._matched_competencies_count || 0)
}));

const byCompetency = {};

for (const asset of learningCatalog) {
  for (const competencyId of asset.competencies) {
    if (!byCompetency[competencyId]) byCompetency[competencyId] = [];
    byCompetency[competencyId].push(asset);
  }
}

for (const key of Object.keys(byCompetency)) {
  byCompetency[key].sort(
    (a, b) => (b.matched_competencies_count || 0) - (a.matched_competencies_count || 0)
  );
}

return [{
  json: {
    ...$json,
    learning_catalog: learningCatalog,
    learning_assets_by_competency: byCompetency
  }
}];
```

# 8. `Code / Build Synthetic Evidence From Matrix`

Mode: `Run Once for All Items`

Эта нода нужна, чтобы старый pipeline не ломался, если `employee_issues` пустой.
В твоем файле `employee_issues` сейчас пустой массив 

То есть мы создаем “нулевое evidence”, но не притворяемся performance-данными.

```javascript
const employeeIssues = $json.employee_issues || [];
const competencies = $json.unified_competency_catalog?.competencies || [];

const assessmentMode = employeeIssues.length ? 'matrix_plus_issues' : 'matrix_only';

const syntheticEvidence = competencies.map(c => ({
  competency_id: c.competency_id,
  evidence_type: 'matrix_definition',
  source: 'matrix_bundle_v1',
  issue_count: employeeIssues.length,
  matrix_levels_available: c.levels.map(l => l.level),
  factual_performance_evidence: false
}));

return [{
  json: {
    ...$json,
    assessment_mode: assessmentMode,
    performance_inference_allowed: employeeIssues.length > 0,
    development_inference_allowed: true,
    synthetic_evidence: syntheticEvidence
  }
}];
```

# 10. Что поменять в старом workflow после adapter layer

Чтобы новый формат реально заработал “в том числе”.

## A. В `Code / Normalize Jira Evidence`

Добавь fallback:

* если `issues` пустые
* но есть `normalized.synthetic_evidence`
* то не падать, а собрать baseline evidence

Пример минимальной логики:

```javascript
const issues = $json.issues || $json.employee_issues || [];
const synthetic = $json.normalized?.synthetic_evidence || [];

if (!issues.length && synthetic.length) {
  return [{
    json: {
      ...$json,
      normalized_evidence: synthetic.map(x => ({
        competency_id: x.competency_id,
        source: x.source,
        evidence_type: x.evidence_type,
        strength: 'low',
        factual_performance_evidence: false
      }))
    }
  }];
}

// иначе старый нормализатор
return items;
```

# 14. Мой совет по внедрению

Сначала внедри только эти ноды:

1. `Detect Input Format`
2. `Normalize Matrix Bundle`
3. `Pick Employee`
4. `Resolve Role Matrix`
5. `Build Unified Competency Catalog`
6. `Build Learning Assets Index`
7. `Build Synthetic Evidence From Matrix`
8. `Build Unified Root Contract`

И уже после этого подключай их к существующему `Build Competency Payloads`.

Так ты быстро проверишь, что:

* сотрудник корректно выбирается
* матрица корректно находится
* learning assets нормально матчятся
* старый workflow не ломается

Если хочешь, следующим сообщением я соберу тебе еще **вторую пачку code nodes**: именно адаптированные версии `Build Competency Payloads`, `Build IDP Payload` и `Build One2One Payload` под `matrix_only` режим.
