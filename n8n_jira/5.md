Вот **эталонный JSON input для WF30** на нескольких компетенциях. Его можно брать как шаблон для n8n.

Он уже рассчитан на нормальный pipeline:

* есть сотрудник
* есть нормализованный каталог компетенций
* есть самооценка
* есть Jira evidence
* есть target levels
* есть все поля для scoring и для LLM

Структура матрицы в исходном файле действительно позволяет это сделать, потому что там компетенции идут с уровнями 0–4 внутри ролей и групп

## Что я бы рекомендовал тебе как практический стандарт

Для реальной работы в n8n я бы оставил именно такие обязательные поля:

### обязательно

* `employee.employee_id`
* `employee.role_id`
* `employee.grade_id`
* `competency_catalog.competencies[].competency_id`
* `competency_catalog.competencies[].skill_type`
* `competency_catalog.competencies[].priority_weight`
* `assessment_input.self_assessments[].competency_id`
* `assessment_input.self_assessments[].self_level`
* `assessment_input.self_assessments[].target_level`
* `evidence_input.jira_competency_evidence[].competency_id`

### желательно

* `comment`
* `criticality`
* `relevance`
* `evidence_strength`
