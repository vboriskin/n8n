# Важное правило по ключам

Во всех таблицах добавь поле:

* `id` — для тикета
* `original` — для canonical tables
* `team` / `root_cause_canonical` / `problem_theme_canonical` / `initiator_department` — для agg tables

Если в твоей Data Table есть отдельный internal row id, всё равно храни свои бизнес-ключи отдельными полями.

# Чем заменять Postgres-ноды

## Вместо `Insert`

используй `Data Table`

* **Operation:** `Create rows`

## Вместо `Upsert`

если в твоей версии нет нормального upsert:

1. `Get many rows` по ключу
2. `If`
3. если найдено → `Update rows`
4. если не найдено → `Create rows`

## Вместо `Select / Execute Query`

используй:

* `Get many rows`
* дальше `Code` для фильтрации/агрегации

## Новый узел после `select_pending_llm`

## 2a. `Code — sort_and_limit_pending`

**Node:** `Code`
**Mode:** `Run Once for All Items`

```javascript
function prio(v) {
  return v === 'high' ? 0 : 1;
}

const sorted = items
  .map(i => i.json)
  .sort((a, b) => {
    if (prio(a.llm_priority) !== prio(b.llm_priority)) {
      return prio(a.llm_priority) - prio(b.llm_priority);
    }
    if ((Number(b.rework_count || 0)) !== (Number(a.rework_count || 0))) {
      return Number(b.rework_count || 0) - Number(a.rework_count || 0);
    }
    const ao = a.overdue_flag === 'yes' ? 1 : 0;
    const bo = b.overdue_flag === 'yes' ? 1 : 0;
    return bo - ao;
  })
  .slice(0, 300);

return sorted.map(r => ({ json: r }));
```

## 15. `aggregate_team_metrics`

### 15a. `Data Table — get_all_tickets_final`

**Table:** `tickets_final`

### 15b. `Code — aggregate_team_metrics`

```javascript
function num(v) {
  const x = Number(v);
  return Number.isFinite(x) ? x : 0;
}

const groups = {};

for (const item of items) {
  const r = item.json;
  const team = r.team_canonical || r.team || 'unknown';

  if (!groups[team]) {
    groups[team] = {
      team,
      tickets_total: 0,
      team_fault_count: 0,
      high_conf_fault_count: 0,
      overdue_count: 0,
      rework_count: 0,
      failure_demand_count: 0,
      avoidable_count: 0,
      automation_candidate_count: 0,
      system_or_process_issue_count: 0,
      durations: []
    };
  }

  const g = groups[team];
  g.tickets_total += 1;

  if (['yes','partial'].includes(r.team_fault_flag)) g.team_fault_count += 1;
  if (r.team_fault_flag === 'yes' && num(r.confidence) >= 0.85) g.high_conf_fault_count += 1;
  if (r.overdue_flag === 'yes') g.overdue_count += 1;
  if (num(r.rework_count) > 0 || r.rework_flag === 'yes') g.rework_count += 1;
  if (r.failure_demand_flag === 'yes') g.failure_demand_count += 1;
  if (r.avoidable_flag === 'yes') g.avoidable_count += 1;
  if (r.automation_candidate === 'yes') g.automation_candidate_count += 1;
  if (['system','process','integration'].includes(r.root_cause_group)) g.system_or_process_issue_count += 1;

  if (r.total_duration_hours !== null && r.total_duration_hours !== undefined) {
    g.durations.push(num(r.total_duration_hours));
  }
}

return Object.values(groups).map(g => {
  const avgDuration = g.durations.length
    ? g.durations.reduce((a,b) => a+b, 0) / g.durations.length
    : 0;

  const team_fault_rate = g.tickets_total ? g.team_fault_count / g.tickets_total : 0;
  const high_conf_fault_rate = g.tickets_total ? g.high_conf_fault_count / g.tickets_total : 0;
  const overdue_rate = g.tickets_total ? g.overdue_count / g.tickets_total : 0;
  const rework_rate = g.tickets_total ? g.rework_count / g.tickets_total : 0;
  const failure_demand_rate = g.tickets_total ? g.failure_demand_count / g.tickets_total : 0;
  const avoidable_rate = g.tickets_total ? g.avoidable_count / g.tickets_total : 0;
  const automation_candidate_rate = g.tickets_total ? g.automation_candidate_count / g.tickets_total : 0;
  const system_or_process_issue_rate = g.tickets_total ? g.system_or_process_issue_count / g.tickets_total : 0;

  const team_problem_score =
    0.25 * high_conf_fault_rate +
    0.15 * rework_rate +
    0.15 * overdue_rate +
    0.15 * failure_demand_rate +
    0.10 * avoidable_rate +
    0.10 * Math.min(avgDuration / 72, 1) +
    0.10 * system_or_process_issue_rate;

  return {
    json: {
      team: g.team,
      tickets_total: g.tickets_total,
      team_fault_rate: Number(team_fault_rate.toFixed(4)),
      high_conf_fault_rate: Number(high_conf_fault_rate.toFixed(4)),
      overdue_rate: Number(overdue_rate.toFixed(4)),
      rework_rate: Number(rework_rate.toFixed(4)),
      failure_demand_rate: Number(failure_demand_rate.toFixed(4)),
      avoidable_rate: Number(avoidable_rate.toFixed(4)),
      automation_candidate_rate: Number(automation_candidate_rate.toFixed(4)),
      avg_duration_hours: Number(avgDuration.toFixed(2)),
      system_or_process_issue_rate: Number(system_or_process_issue_rate.toFixed(4)),
      team_problem_score: Number(team_problem_score.toFixed(4))
    }
  };
});
```

# Универсальный шаблон Upsert для Data Table

Используй везде, где нужен upsert.

## Шаги

1. `Data Table — Get many rows`

   * filter by key
2. `If`

   * если найдено
3. `Data Table — Update rows`
4. `Data Table — Create rows`

## Пример для `tickets_final`

### `Get many rows`

Filter:

* `id = {{$json.id}}`

### `If`

Condition:

* `{{$items().length}} > 0`

### `Update rows`

Обновляй найденную запись

### `Create rows`

Создавай новую запись

# Что упрощается

С `Data Table` лучше делать так:

* не пытаться эмулировать SQL join в одном узле
* читать таблицы целиком или выборками
* merge/join/aggregation делать в `Code`
