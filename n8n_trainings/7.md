Неплохой промпт уже — он строгий и хорошо ограничивает модель. Но для GigaChat (и вообще не-OpenAI моделей) можно заметно повысить точность за счёт **структурирования мышления + устранения неоднозначностей + защиты от “галлюцинаций списка”**.

Ниже — улучшенная версия + объяснение, что именно усилили.

## 🔧 Улучшенный промпт (переработанный)

```
Ты — система строгого сопоставления учебных курсов с компетенциями.

Твоя задача — выбрать ТОЛЬКО те компетенции, которые напрямую и явно соответствуют обучению.

=====================
ДОПУСТИМЫЕ КОМПЕТЕНЦИИ
=====================
Используй ТОЛЬКО этот список. Нельзя изменять формулировки, обобщать или придумывать новые.

{{ JSON.stringify($json.competencies) }}

=====================
ДАННЫЕ ОБУЧЕНИЯ
=====================
ID: {{ $json.id }}
Название: {{ $json.name }}
Описание: {{ $json.description || "Нет описания" }}
Тип: {{ $json.type || "Не указан" }}
Длительность: {{ $json.duration || "Не указана" }}

=====================
АЛГОРИТМ РАБОТЫ
=====================
1. Определи ключевые темы обучения (мысленно, не выводи).
2. Сравни темы ТОЛЬКО с формулировками компетенций.
3. Выбирай компетенцию, только если:
   - она явно покрывается содержанием обучения
   - нет двусмысленности
4. Если есть сомнение — НЕ добавляй компетенцию.

=====================
ОЦЕНКА УВЕРЕННОСТИ
=====================
Назначай score строго по правилам:

- 0.9–1.0 → компетенция прямо указана в названии или описании
- 0.8–0.9 → курс в основном про эту компетенцию
- 0.7–0.8 → компетенция — значимая часть курса

Запрещено:
- использовать score < 0.7
- завышать score без явного основания

=====================
ФОРМАТ ВЫВОДА
=====================
- Только JSON
- Без markdown
- Без комментариев
- Без лишних полей

reason:
- максимум 5 слов
- только факт (без воды)

=====================
РЕЗУЛЬТАТ
=====================

Если есть совпадения:
{
  "id": "{{ $json.id }}",
  "name": "{{ $json.name }}",
  "matched_competencies": [
    {
      "name": "точное название из списка",
      "score": 0.85,
      "reason": "краткая причина"
    }
  ]
}

Если совпадений нет:
{
  "id": "{{ $json.id }}",
  "name": "{{ $json.name }}",
  "matched_competencies": []
}
```

## 🚀 Что стало лучше (ключевые улучшения)

### 1. Добавлен “АЛГОРИТМ РАБОТЫ”

GigaChat сильно выигрывает от явного пошагового процесса.

👉 Это уменьшает:

* случайные совпадения
* “угадывание”
* лишние компетенции

---

### 2. Жёсткий анти-галлюцинационный блок

```
Нельзя изменять формулировки, обобщать или придумывать новые
```

👉 Это критично — иначе модель начинает:

* перефразировать компетенции
* выбирать “похожие”, но не те

---

### 3. Усилен критерий “сомневаешься — не добавляй”

Это снижает переизбыточные матчи (главная проблема таких задач).

---

### 4. Убрана размытость в score

Теперь:

* каждое значение связано с конкретным условием
* есть запрет на “завышение”

---

### 5. Скрытое рассуждение (мысленно)

```
(мысленно, не выводи)
```

👉 Это:

* улучшает точность
* не ломает JSON

## 💡 Концептуальные улучшения (если хочешь ещё сильнее)

Если без эмбеддингов, вот что реально бустит качество:

---

### ✅ 1. Добавить "негативные примеры" (очень мощно)

В конец промпта:

```
ПРИМЕРЫ ОШИБОК:

Нельзя:
- выбирать компетенцию только по одному слову
- выбирать слишком общую компетенцию
- выбирать компетенцию, если тема лишь частично затронута
```

---

### ✅ 2. Ограничить количество компетенций

Если это допустимо:

```
Максимум 5 компетенций на курс
```

👉 резко снижает шум

---

### ✅ 3. Сортировка по score (важно для downstream)

```
Отсортируй matched_competencies по убыванию score
```

---

### ✅ 4. Усиление точности через "пересечение смыслов"

Можно добавить:

```
Компетенция должна покрывать основную цель курса, а не отдельный модуль.
```

## ⚠️ Главная проблема твоего текущего промпта

Он **слишком “доверяет” модели**, а не:

* ограничивает
* направляет
* заставляет фильтровать

Мой вариант делает модель более:
👉 строгой
👉 “консервативной”
👉 предсказуемой
