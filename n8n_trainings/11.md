Ниже — практический skeleton для n8n по нодам. Не экспорт workflow JSON, а именно **рабочая схема**, которую удобно собрать руками и адаптировать под свой стек.

Сделаю в формате:

* какие ноды
* что каждая делает
* какие поля на входе/выходе
* какие IF-ветки
* какие expression вставлять

---

# Общая схема пайплайна

```text
Trigger / Source
  ↓
Normalize Input
  ↓
Build LLM Prompt
  ↓
LLM Pass 1
  ↓
Extract Raw Response
  ↓
Validate Pass 1
  ↓
IF validation_status == valid
  ├─ yes → Save Final
  └─ no  → Build Retry Prompt
              ↓
           LLM Pass 2
              ↓
           Extract Retry Response
              ↓
           Validate Pass 2
              ↓
           IF validation_status == valid
              ├─ yes → Save Final
              └─ no  → Manual Review / Dead Letter
```

# Node 2 — Normalize Input

Тип: `Code`

Задача:

* почистить строки
* убрать HTML
* нормализовать null
* подготовить безопасные поля

### Код

```javascript
function cleanText(value, fallback = '') {
  let text = String(value ?? fallback);

  text = text.replace(/<[^>]*>/g, ' ');
  text = text.replace(/&nbsp;/g, ' ');
  text = text.replace(/\s+/g, ' ').trim();

  return text || fallback;
}

return items.map(item => {
  const j = item.json;

  const id = j.id ?? '';
  const name = cleanText(j.name, 'Без названия');
  const description = cleanText(j.description, 'Нет описания');
  const type = cleanText(j.type, 'Не указан');
  const duration = cleanText(j.duration, 'Не указана');

  const competencies = Array.isArray(j.competencies)
    ? j.competencies
        .map(x => String(x ?? '').trim())
        .filter(Boolean)
    : [];

  return {
    json: {
      ...j,
      id,
      name,
      description,
      type,
      duration,
      competencies,
      name_clean: name,
      description_clean: description,
      type_clean: type,
      duration_clean: duration,
    },
  };
});
```

### На выходе

Появятся:

* `name_clean`
* `description_clean`
* `type_clean`
* `duration_clean`

# Node 3 — Build LLM Prompt

Тип: `Set` или `Code`

Проще через `Set`.

Добавь поля:

* `system_prompt`
* `user_prompt`
* `prompt_version`
* `validation_pass`

### `prompt_version`

```text
competency_matcher_v1
```

### `validation_pass`

```text
1
```

### `system_prompt`

Вставь тот system prompt, который мы уже собрали раньше.

### `user_prompt`

Пример expression:

```javascript
={{
`Определи соответствие курса компетенциям.

Допустимые компетенции:
${JSON.stringify($json.competencies)}

Данные курса:
{
  "id": ${JSON.stringify($json.id)},
  "name": ${JSON.stringify($json.name_clean)},
  "description": ${JSON.stringify($json.description_clean)},
  "type": ${JSON.stringify($json.type_clean)},
  "duration": ${JSON.stringify($json.duration_clean)}
}

Верни только JSON в формате:
{
  "id": ${JSON.stringify($json.id)},
  "name": ${JSON.stringify($json.name_clean)},
  "matched_competencies": [
    {
      "name": "точное название компетенции из списка",
      "score": 0.85,
      "reason": "краткая причина"
    }
  ]
}`
}}
```

# Node 4 — LLM Pass 1

Тип зависит от интеграции:

* OpenAI Chat Model
* HTTP Request к GigaChat
* AI Agent node
* Basic LLM Chain

Для production лучше, чтобы на выходе было одно поле с сырым текстом ответа.

Например:

* `response`
* `content`
* `message`
* `data.choices[0].message.content`

# Node 5 — Extract Raw Response

Тип: `Set` или `Code`

Задача: привести ответ модели к одному полю `llm_response`.

Если у тебя OpenAI-like формат, можно через `Set`:

### `llm_response`

```javascript
={{ $json.content || $json.response || $json.message || $json.text || '' }}
```

Если ответ вложенный, лучше через `Code`.

### Код

```javascript
return items.map(item => {
  const j = item.json;

  const llm_response =
    j.content ??
    j.response ??
    j.message ??
    j.text ??
    j?.data?.choices?.[0]?.message?.content ??
    '';

  return {
    json: {
      ...j,
      llm_response,
    },
  };
});
```

# Node 9 — Build Retry Prompt

Тип: `Set`

Поля:

* `validation_pass = 2`
* `retry_system_prompt`
* `retry_user_prompt`

### `validation_pass`

```text
2
```

### `retry_system_prompt`

Можно оставить тот же system prompt, либо сделать более жёсткий.

### `retry_user_prompt`

Expression:

```javascript
={{
`Проверь классификацию курса заново и верни исправленный результат.

Допустимые компетенции:
${JSON.stringify($json.retry_prompt_context.competencies)}

Данные курса:
{
  "id": ${JSON.stringify($json.retry_prompt_context.id)},
  "name": ${JSON.stringify($json.retry_prompt_context.name)},
  "description": ${JSON.stringify($json.retry_prompt_context.description)},
  "type": ${JSON.stringify($json.retry_prompt_context.type)},
  "duration": ${JSON.stringify($json.retry_prompt_context.duration)}
}

Предыдущий ответ модели:
${JSON.stringify($json.retry_prompt_context.previous_llm_response)}

Ошибки валидации:
${JSON.stringify($json.retry_prompt_context.validation_errors)}

Предупреждения:
${JSON.stringify($json.retry_prompt_context.validation_warnings)}

Причины повторной проверки:
${JSON.stringify($json.retry_prompt_context.suspicious_reasons)}

Требования:
- Используй только компетенции из допустимого списка
- Удаляй всё, что не подтверждается напрямую названием или описанием
- Удаляй всё, что относится не к основной теме курса
- Если есть сомнение, исключай компетенцию
- Максимум 5 компетенций
- Не добавляй дубликаты
- Отсортируй matched_competencies по убыванию score
- Все score должны быть числами от 0.70 до 1.00
- reason максимум 5 слов

Верни только валидный JSON.`
}}
```

# Node 15 — Manual Review / Dead Letter

Сюда складывай проблемные записи.

Полезно сохранять:

* исходные данные курса
* первый ответ модели
* второй ответ модели
* ошибки валидации
* suspicious reasons
* prompt version

### Поля для сохранения

Например через `Set`:

```javascript
={{
{
  "id": $json.id,

  "description": $json.description,
  "type": $json.type,
  "duration": $json.duration,
  "competencies": $json.competencies,
  "validation_status": $json.validation_status,
  "validation_errors": $json.validation_errors,

  "suspicious_reasons": $json.suspicious_reasons,
  "previous_llm_response": $json.retry_prompt_context?.previous_llm_response || $json.llm_response,
  "retry_llm_response": $json.retry_llm_response || null,
```

# Самая практичная конфигурация IF-логики

## Первый IF

```javascript
={{ $json.validation_status === 'valid' }}
```

## Альтернатива: отдельно invalid и retry_needed

Можно сделать 2 IF:

### IF 1

```javascript
={{ $json.validation_status === 'invalid' }}
```

→ всегда retry

### IF 2

```javascript
={{ $json.validation_status === 'retry_needed' }}
```

→ retry only suspicious

### ELSE

→ final save

Это полезнее, если хочешь по-разному логировать проблемы.

# Если нужен batch

Для массового прогона обычно добавляют:

## Перед Normalize

`Split In Batches`

Например по 20–100 записей, в зависимости от лимитов API.

Схема:

```text
Source
  ↓
Split In Batches
  ↓
Normalize
  ↓

  ↓

  ↓
```

```text
  ↓
```

# Маленький, но важный совет

Для n8n очень удобно сделать ещё одну `Code` node в самом конце, которая приводит финальный объект к единому формату независимо от того, был это pass 1 или pass 2.

Например:

```javascript
return items.map(item => {
  const j = item.json;

  return {
    json: {
      course_id: j.id,
      course_name: j.name,
      final_result: j.parsed_result ?? null,
      final_status: j.final_status ?? j.validation_status,
      prompt_version: j.prompt_version ?? 'unknown',
      validation_pass: j.validation_pass ?? null,
      manual_review_required: j.manual_review_required ?? false,
      processed_at: new Date().toISOString(),
    },
  };
});
```

Это потом сильно упрощает загрузку в БД.
