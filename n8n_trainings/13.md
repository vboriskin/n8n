## 2. Универсальный extractor после OpenAI node

`Code` node:

```javascript
function extractOpenAIText(j) {
  const candidates = [
    j.output_text,
    j.text,
    j.content,
    j.message,
    j.response,
    j?.choices?.[0]?.message?.content,
    j?.data?.choices?.[0]?.message?.content,
    j?.messages?.[0]?.content,
  ];

  for (const candidate of candidates) {
    if (typeof candidate === 'string' && candidate.trim()) {
      return candidate.trim();
    }

    if (Array.isArray(candidate) && candidate.length) {
      const joined = candidate.map(x => {
        if (typeof x === 'string') return x;
        if (x?.text) return x.text;
        if (x?.content) return x.content;
        return '';
      }).filter(Boolean).join('\n').trim();

      if (joined) return joined;
    }
  }

  return '';
}

return items.map(item => {
  const j = item.json;
  const llm_response = extractOpenAIText(j);

  return {
    json: {
      ...j,
      llm_response,
      extraction_failed: !llm_response,
    },
  };
});
```

## 3. Если extractor ничего не нашёл

Добавь сразу после него `IF`:

Условие:

```javascript
={{ !$json.llm_response }}
```

Если `true`, отправляй в отдельную ветку `Manual Review / Debug`.

Туда сохраняй весь raw output:

`Set` node:

* `debug_raw_output = {{$json}}`
* `final_status = extraction_failed`
* `manual_review_required = true`

Так ты не потеряешь запись и потом быстро поймёшь, где лежит текст ответа.

```text
Ты — строгий классификатор соответствия обучения и компетенций.

Верни только валидный JSON.
Без markdown.
Без комментариев.

Не придумывай новые компетенции.

Если есть сомнение — не добавляй компетенцию.
```

## 6. Что делать потом, когда сможешь посмотреть output OpenAI node

Когда будет возможность, просто открой output OpenAI node и проверь, есть ли одно из полей:

* `output_text`
* `content`
* `text`
* `message`
* `choices[0].message.content`

После этого можно будет сократить extractor до 2–3 строк.

Вот шаблон упрощённого extractor, который потом заменишь:

```javascript
return items.map(item => ({
  json: {
    ...item.json,
    llm_response: item.json.content || ''
  }
}));
```

## 7. Один практический совет

Чтобы быстрее отладить pipeline, сначала прогони 3 тестовых записи:

* курс с очевидным совпадением
* курс с 2–3 совпадениями
* курс без совпадений

Так ты сразу увидишь:

* как OpenAI node реально кладёт ответ
* как отрабатывает validator
* нужен ли второй проход

Могу сразу дать тебе ещё и **упрощённую версию validator-а для MVP**, чтобы сначала запустить систему без лишней строгости.
