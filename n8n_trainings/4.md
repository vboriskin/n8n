Ниже — **production-вариант workflow для n8n** без `HTTP Request`, только с **OpenAI node**.

Что добавлено по сравнению с прошлой версией:

* `Webhook` на вход
* обработка **массива курсов**
* `Split In Batches`
* fallback shortlist
* repair step для битого JSON
* финальная строгая валидация
* единый стабильный выход

Ожидаемый вход в webhook:

```json
{
  "courses": [
    {
      "id": "course-001",
      "name": "Основы Python для анализа данных",
      "description": "Изучение синтаксиса Python, переменных, функций, циклов, pandas и базовой обработки данных.",
      "type": "online",
      "duration": "24 часа",
      "competencies": [
        "Python",
        "Программирование",
        "Анализ данных",
        "SQL",
        "Управление проектами",
        "Информационная безопасность",
        "Деловая коммуникация"
      ]
    }
  ]
}
```

### Что нужно поменять после импорта

Заменить в обеих OpenAI-нодах:

* `YOUR_OPENAI_CREDENTIAL_ID`
* при необходимости имя credentials

### Что делает каждая нода

`Webhook`
Принимает POST с одним курсом или массивом `courses`.

`Explode Courses`
Если пришёл массив — превращает его в отдельные items.

`Split In Batches`
Обрабатывает по одному курсу за раз.

`Prepare Course Data`
Нормализует текст, режет шум, собирает shortlist.

`OpenAI Match Competencies`
Основной strict matching prompt.

`Parse LLM Output`
Пытается распарсить ответ модели.

`Needs Repair?`
Проверяет, нужен ли repair.

`OpenAI Repair JSON`
Чинит JSON, если модель вернула мусор.

`Validate Result`
Оставляет только допустимые компетенции, score >= 0.7, убирает дубли.

`Collect Results`
Возвращает итог:

```json
{
  "results": [
    {
      "id": "...",
      "name": "...",
      "matched_competencies": [...]
    }
  ]
}
```

### Практические замечания

У этого JSON есть тонкость: в разных версиях n8n `Merge` и `Split In Batches` иногда ведут себя чуть по-разному. Если после импорта увидишь, что цикл отрабатывает нестабильно, самый простой и надёжный прод-вариант — вообще **не использовать `Split In Batches`**, а обработать весь вход в одном `Code` через `return courses.map(...)`, а потом собирать результат обратно. Это часто даже стабильнее.

### Я бы рекомендовал ещё более надёжный production-вариант

Самая практичная версия для n8n обычно такая:

`Webhook → Explode Courses → Prepare Course Data → OpenAI Match → Parse/Repair → Validate → Respond`

То есть без `Split In Batches`, если не нужен контроль поштучной нагрузки.
