Хороший базовый промпт — уже достаточно строгий. Но его можно усилить **без эмбеддингов** за счёт трёх вещей:

1. уменьшить «галлюцинации»,
2. сделать выбор более дискретным,
3. заставить модель *думать через фильтрацию*, а не сразу генерить ответ.

Ниже — улучшенная версия + объяснение, что именно поменялось.

---

## 🔧 Улучшенный промпт

```text
Ты — эксперт по строгому сопоставлению учебных курсов с компетенциями.

Твоя задача — выбрать ТОЛЬКО те компетенции, которые напрямую и явно соответствуют обучению.

Используй ТОЛЬКО компетенции из списка. Запрещено:
- придумывать новые компетенции
- перефразировать названия
- выбирать «похожие по смыслу»

Список допустимых компетенций:
{{ JSON.stringify($json.competencies) }}

Данные обучения:
- ID: {{ $json.id }}
- Название: {{ $json.name }}
- Описание: {{ $json.description || "Нет описания" }}
- Тип: {{ $json.type || "Не указан" }}
- Длительность: {{ $json.duration || "Не указана" }}

АЛГОРИТМ РАБОТЫ:

Шаг 1. Определи ключевые темы обучения (мысленно, не выводи).
Шаг 2. Для каждой компетенции из списка проверь:
  - есть ли прямое совпадение по теме
  - упоминается ли это явно в названии или описании
Шаг 3. Отбрось:
  - косвенные связи
  - общие или «подходящие по смыслу» компетенции
  - слишком широкие категории
Шаг 4. Оставь только однозначные соответствия.

ПРАВИЛА ОЦЕНКИ (score):
- 0.7–0.8 — компетенция явно покрывается, но не основная
- 0.8–0.9 — одна из ключевых тем обучения
- 0.9–1.0 — прямо указана в названии или описании

ВАЖНО:
- Не добавляй компетенцию, если score < 0.7
- Максимум 5 компетенций
- Если есть сомнения — НЕ добавляй компетенцию
- Лучше вернуть пустой список, чем ошибиться

reason:
- не более 5 слов
- только конкретика (без воды)

ФОРМАТ ОТВЕТА — строго JSON, без markdown и пояснений:

Если есть совпадения:
{
  "id": "{{ $json.id }}",
  "name": "{{ $json.name }}",
  "matched_competencies": [
    {
      "name": "точное название из списка",
      "score": 0.85,
      "reason": "краткая причина"
    }
  ]
}

Если совпадений нет:
{
  "id": "{{ $json.id }}",
  "name": "{{ $json.name }}",
  "matched_competencies": []
}
```

## 💡 Что это улучшает

### 1. Убирает «семантическое размывание»

Добавлено:

> запрещено выбирать «похожие по смыслу»

LLM очень любит «дотягивать» соответствия — это главный источник шума.

---

### 2. Вводит **жёсткий pipeline (алгоритм)**

LLM лучше работает, когда:

* сначала **анализ**,
* потом **фильтрация**,
* потом **выбор**.

Это снижает случайные попадания.

---

### 3. Добавлен bias к precision > recall

Фраза:

> Лучше вернуть пустой список, чем ошибиться

Это критично, если тебе важна чистота данных.

---

### 4. Ограничение на количество (max 5)

Без этого модель:

* начинает «накидывать всё подряд»
* особенно на длинных списках

---

### 5. Ужесточение определения «явной связи»

Теперь модель должна:

* искать **прямое совпадение темы**
* а не «логически связанное»

## 🚀 Дополнительные идеи (без эмбеддингов)

Если хочешь ещё сильнее улучшить:

### 🔹 1. Добавь анти-примеры (очень эффективно)

Прямо в промпт:

```text
ПРИМЕРЫ НЕПРАВИЛЬНОГО СОПОСТАВЛЕНИЯ:
- Курс "Основы Excel" → "Аналитическое мышление" (слишком общее)
- Курс "Коммуникации" → "Лидерство" (косвенная связь)
```

Это резко снижает галлюцинации.

---

### 🔹 2. Введи «порог явности»

```text
Компетенция считается подходящей только если:
- явно упоминается ИЛИ
- является основной темой курса
```

---

### 🔹 3. (Сильный приём) — принудительная проверка

Добавь:

```text
Перед добавлением компетенции задай себе вопрос:
"Эта компетенция прямо следует из текста?"
Если нет — не добавляй.
```

---

### 🔹 4. Сортировка по score

Если важно качество:

```text
Отсортируй matched_competencies по убыванию score
```

---

## ⚠️ Типичная проблема GigaChat (и не только)

Он может:

* игнорировать список
* слегка менять названия

Если это происходит — добавь:

```text
Название компетенции должно совпадать СИМВОЛ В СИМВОЛ со списком
```

---

Если хочешь, могу:

* адаптировать под твой конкретный формат компетенций
* или сделать **двухшаговый пайплайн (candidate → re-rank)** без эмбеддингов — это даёт ещё +20–30% к качеству
