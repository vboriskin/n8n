Ниже готовый **n8n workflow JSON** без `HTTP Request`, с использованием **OpenAI node**.

Что делает workflow:

1. **Set Input Example** — пример входных данных
2. **Prepare Course Data** — нормализация + shortlist компетенций
3. **OpenAI Match Competencies** — строгий prompt для сопоставления
4. **Validate Result** — очистка и валидация JSON ответа

## Что нужно поменять у себя

### 1. Credentials

В ноде `OpenAI Match Competencies` замени:

* `YOUR_OPENAI_CREDENTIAL_ID`
* при необходимости имя credentials

### 2. Модель

Сейчас стоит:

* `gpt-4.1-mini`

Можно заменить на:

* `gpt-4.1`
* `gpt-4o-mini`
* ту модель, которая у тебя доступна в OpenAI node

### 3. Источник данных

Сейчас первая нода — это просто пример.

В бою обычно вместо `Set Input Example` ставят:

* `Webhook`
* `Google Sheets`
* `Postgres`
* `Airtable`
* `Manual Trigger + Set`
* или любой другой источник

Главное, чтобы на вход приходило:

```json
{
  "id": "course-001",
  "name": "Название курса",
  "description": "Описание курса",
  "type": "Тип",
  "duration": "Длительность",
  "competencies": ["Компетенция 1", "Компетенция 2"]
}
```

## Важное замечание по OpenAI node в n8n

У разных версий n8n/OpenAI node немного отличаются поля ответа.
Если у тебя `Validate Result` не увидит ответ модели, пришли мне экспорт именно твоей OpenAI-ноды или скрин полей output — я сразу дам точную версию JS под твою сборку n8n.

## Что я бы ещё улучшил в следующей версии

Я бы сделал ещё 2 апгрейда:

### Вариант A — fallback при пустом shortlist

Сейчас это уже частично есть, но можно сделать лучше:

* если shortlist пустой, брать top 30 из full list
* или весь full list, если он короткий

### Вариант B — retry repair

Если модель вернула битый JSON:

* вторая OpenAI нода “исправь только JSON”
* и потом снова validate

Это заметно повышает стабильность на больших потоках.

## Рекомендация для production

Для продакшна я бы держал такую схему:

`Source → Prepare Course Data → OpenAI Match → Validate Result`

А если качество нужно ещё выше:

`Source → Prepare Course Data → OpenAI Extract Topics → OpenAI Match → Validate Result`

То есть 2 LLM-прохода вместо одного.
