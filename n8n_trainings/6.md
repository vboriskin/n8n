Хороший базовый промпт — уже довольно строгий. Но без эмбеддингов основная проблема тут не в «формулировке», а в **снижении галлюцинаций + повышении точности отбора**. Ниже — концептуальные улучшения и затем готовая улучшенная версия.

---

## 🔧 Ключевые идеи улучшения

### 1. Явно запретить "додумывание"

Сейчас модель может «дотягивать» связь. Нужно прямо сказать:

> если нет прямого текстового подтверждения — не сопоставлять

---

### 2. Ввести *внутренний этап проверки* (скрытый reasoning)

Даже без CoT можно заставить модель:

* сначала найти **фрагмент текста**
* потом принять решение

👉 Это резко повышает точность даже без эмбеддингов

---

### 3. Нормализовать компетенции (важно)

LLM путается, если список большой. Добавь:

* сравнение **по смыслу, но с привязкой к формулировке**
* запрет на "расширительное толкование"

---

### 4. Ограничить количество результатов

Без этого модель часто «накидывает лишнего»

---

### 5. Добавить правило "приоритет точности над полнотой"

Это критично для твоего кейса

---

### 6. Уточнить шкалу score (сейчас слегка размыта)

Добавим чёткие критерии

---

## 🚀 Улучшенный промпт

Вот версия, которую можно прямо использовать:

```
Ты — эксперт по строгому сопоставлению учебных курсов с компетенциями.

Твоя задача — находить ТОЛЬКО явные и однозначные соответствия.

Используй только компетенции из списка. Нельзя добавлять или изменять формулировки.

Список компетенций:
{{ JSON.stringify($json.competencies) }}

Данные обучения:
- ID: {{ $json.id }}
- Название: {{ $json.name }}
- Описание: {{ $json.description || "Нет описания" }}
- Тип: {{ $json.type || "Не указан" }}
- Длительность: {{ $json.duration || "Не указана" }}

---

ПРАВИЛА ОТБОРА:

1. Добавляй компетенцию ТОЛЬКО если:
   - она прямо следует из названия ИЛИ описания
   - или явно упоминается по смыслу без расширительной интерпретации

2. НЕ добавляй компетенцию если:
   - требуется догадка или интерпретация
   - связь косвенная или общая
   - компетенция шире, чем тема курса

3. Приоритет: ТОЧНОСТЬ > ПОЛНОТА  
   Лучше вернуть меньше результатов, чем добавить лишние.

4. Максимум 5 компетенций.

---

ОЦЕНКА (score):

- 0.9–1.0 → компетенция явно указана в названии или тексте
- 0.8–0.89 → курс в основном посвящён этой компетенции
- 0.7–0.79 → компетенция — значимая часть курса

НЕ добавляй, если score < 0.7

---

ОБОСНОВАНИЕ:

- reason: максимум 5 слов
- только конкретика (без общих фраз)

---

ВНУТРЕННЯЯ ПРОВЕРКА (обязательно перед ответом):

Для каждой компетенции:
1. Найди подтверждение в тексте (название или описание)
2. Убедись, что связь прямая, без догадок
3. Только после этого добавляй

---

ФОРМАТ ОТВЕТА (строго JSON, без текста и markdown):

Если есть совпадения:
{
  "id": "{{ $json.id }}",
  "name": "{{ $json.name }}",
  "matched_competencies": [
    {
      "name": "точное название из списка",
      "score": 0.85,
      "reason": "краткая причина"
    }
  ]
}

Если совпадений нет:
{
  "id": "{{ $json.id }}",
  "name": "{{ $json.name }}",
  "matched_competencies": []
}
```

> выбрать максимум 10 кандидатов

**Шаг 2** — твой строгий промпт
👉 сильно снижает шум

### 2. Сортировка + отсечение

Добавь правило:

> возвращай только top-N по score

---

### 3. Дедуп / схлопывание

Если компетенции похожи — модель иногда дублирует смысл
Можно добавить:

> избегай семантически дублирующих компетенций

* адаптировать под **конкретный размер списка (10 / 1000 компетенций)**
