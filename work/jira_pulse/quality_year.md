# 🧠 PROMPT 1 — построение выборки и расчёта

```text
Построй расчет метрики "Длительность этапов от Готов к ИФТ до Ожидание внедрения" в Jira.

1. Возьми задачи:
- issuetype = Story  ← ЗАМЕНИТЬ при необходимости
- статус = внедрено в ПРОМ ← ЗАМЕНИТЬ (resolved / deployment / custom field)
- период = последние 90 дней ← ЗАМЕНИТЬ при необходимости

2. Отфильтруй:
- Domain issue in (sigma, delta) ← ЗАМЕНИТЬ на поле/labels
- LT issue > 0 ← ЗАМЕНИТЬ на реальное поле

3. Исключи задачи:
- vendor ← ЗАМЕНИТЬ (labels/components)
- команды СБТ / Run ← ЗАМЕНИТЬ (team field)
- первые 3 релиза ← ЗАМЕНИТЬ (fixVersion / release)
- канареечные релизы (кроме iOS) ← ЗАМЕНИТЬ
- AI-агенты ← ЗАМЕНИТЬ

4. Для каждой задачи посчитай сумму времени в статусах:
- "Ожидание ИФТ" ← ЗАМЕНИТЬ
- "ИФТ" ← ЗАМЕНИТЬ
- "Ожидание ПСИ" ← ЗАМЕНИТЬ
- "ПСИ" ← ЗАМЕНИТЬ

Если статус отсутствовал → время = 0

5. Рассчитай:
Длительность AS IS = сумма времени по этапам

6. Итоговая метрика:
Σ(Длительность AS IS * Коэф(A) + Коэф(B)) / Кол-во Story

Вывести результат по командам.
```

---

## ❗ Что уточнить в Jira (для этого промпта)

* какие реальные названия статусов (ИФТ / ПСИ)
* как определяется "внедрено в ПРОМ"
* есть ли **time in status**
* где хранится:

  * team
  * domain (sigma/delta)
* как помечаются:

  * vendor
  * run / СБТ
  * AI / канареечные релизы
* есть ли fixVersion / release логика

# ⚙️ PROMPT 2 — реализация через инструменты (eazyBI / dashboard)

```text
Реализуй метрику "Длительность этапов от Готов к ИФТ до Ожидание внедрения" в Jira Dashboard (или eazyBI).

1. Создай фильтр Story:
- issuetype = Story ← ЗАМЕНИТЬ
- внедрено в ПРОМ за период ← ЗАМЕНИТЬ
- domain in (sigma, delta) ← ЗАМЕНИТЬ

2. Исключи:
- vendor / external ← ЗАМЕНИТЬ
- команды СБТ / Run ← ЗАМЕНИТЬ
- первые 3 релиза ← ЗАМЕНИТЬ
- канареечные релизы ← ЗАМЕНИТЬ

3. Получи метрики:
- время в статусах:
  - Ожидание ИФТ ← ЗАМЕНИТЬ
  - ИФТ ← ЗАМЕНИТЬ
  - Ожидание ПСИ ← ЗАМЕНИТЬ
  - ПСИ ← ЗАМЕНИТЬ

4. Рассчитай:
Duration = сумма времени по статусам

5. Примени коэффициенты:
Adjusted = Duration * A + B ← ЗАМЕНИТЬ источник A/B

6. Итог:
Metric = SUM(Adjusted) / COUNT(Story)

7. Построй:
- разрез: Team ← ЗАМЕНИТЬ поле
- период: месяц / квартал ← ЗАМЕНИТЬ
- визуализация: среднее значение

Если нет time-in-status:
используй eazyBI или плагин Time in Status.
```

---

## ❗ Что уточнить в Jira (для этого промпта)

* есть ли eazyBI / Time in Status
* где брать коэффициенты A и B
* как называется поле команды
* какие статусы реально используются
* есть ли history (чтобы считать время)
* в каком виде нужен дашборд (по командам / проектам)
