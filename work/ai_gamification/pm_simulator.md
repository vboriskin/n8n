# Требования к игре

## Идея

Это браузерная мини-игра, в которой игрок выступает в роли **project manager’а** и одновременно работает с тремя каналами:

* Jira
* Email
* Messenger

Через эти каналы приходят:

* задачи
* блокеры
* баги
* вопросы от команды
* запросы клиента
* давление от руководства

Игрок должен удерживать проект под контролем.

## Что игрок должен делать

* менять приоритет задач
* менять статус задач
* закрывать задачи
* отвечать на письма
* реагировать на сообщения
* эскалировать проблемы
* откладывать менее важное
* быстро разруливать критические события

## Что должно быть видно без объяснений

Игрок должен сразу понять:

* это симулятор project manager’а
* Jira, почта и чат — три источника нагрузки
* решения влияют на проект
* надо держать баланс между срочным и важным

## Формат проекта

Сделай **чистый MVP Basic+**:

* один экран
* desktop-first
* чистый HTML/CSS/JS
* без библиотек
* без фреймворков
* без сборки
* запуск простым открытием `index.html`

## Основные игровые панели

### Jira

Покажи список задач или мини-канбан.
У задач есть:

* title
* type
* priority
* deadline / urgency
* status

Игрок может:

* повысить/понизить приоритет
* перевести статус
* открыть/закрыть задачу

### Email

Список писем от:

* клиента
* QA
* руководителя
* команды

Для каждого письма нужны быстрые действия:

* Reply
* Snooze
* Escalate
* Ignore

### Messenger

Список коротких срочных сообщений.
Быстрые действия:

* Respond
* Prioritize
* Create task
* Delay

## Метрики

Минимум:

* **Project Health**
* **Chaos**
* **Score**

Допустимы ещё:

* Client Satisfaction
* Team Morale

Но не усложняй без необходимости.

## Раунд

Раунд должен длиться **2–4 минуты**.

## Контент

Нужны:

* 6–8 стартовых задач
* 8–12 событий
* несколько кризисов
* нарастающее давление

## Последствия

Каждое решение должно менять состояние игры.

Примеры:

* ignored critical bug → Health вниз, Chaos вверх
* replied to client fast → Score вверх, Chaos вниз
* wrong close of task → later return from QA
* too many snoozes → backlog pressure
* only reacting to urgent → hidden overdue tasks later

## Победа

* таймер закончился
* Health выше минимального порога
* Chaos ниже критического

## Поражение

* Health = 0
  или
* Chaos достиг максимума

## Экран финала

Покажи:

* итоговый score
* сколько задач завершено
* сколько критичных событий обработано
* короткий титул игрока

Примеры титулов:

* Calm Operator
* Crisis PM
* Burnout Manager
* Delivery Hero

## Визуальный стиль

Сделай интерфейс похожим на знакомые рабочие инструменты, но чуть более игровым:

* аккуратные панели
* карточки
* бейджи
* заметные приоритеты
* таймер
* хорошие визуальные состояния
* не перегружено

## Качество реализации

* не делай статичный макет
* это должна быть реально играбельная мини-игра
* код должен быть чистым и расширяемым
* логика должна быть простой, но живой
* интерфейс должен демонстрировать именно работу PM
