# Анализ плана доработок Q2 (.xlsx)

Локальное standalone-приложение (без npm, без backend, без фреймворков) для анализа Excel-файла с планом доработок по релизам.

## Что делает приложение

- Загружает `.xlsx` через UI (upload).
- Пытается загрузить `plan.xlsx` рядом с `index.html` (если браузер разрешает доступ).
- Собирает единый реестр задач из всех вкладок файла.
- Строит summary:
  - всего задач,
  - в scope (ДА),
  - вне scope (НЕТ),
  - под вопросом (?),
  - в релизе 1.20,
  - в релизе 1.21,
  - без релиза.
- Показывает срез по релизам 1.20 и 1.21:
  - число задач,
  - breakdown по типам работ,
  - breakdown по командам.
- Показывает блок `Без релиза` с фильтрами и таблицей.
- Показывает блок `Под вопросом (scope = ?)`.
- Показывает сравнение по командам + breakdown типов работ.
- Поддерживает глобальные фильтры.
- Экспортирует CSV:
  - текущая выборка всех задач,
  - таблица без релиза,
  - таблица под вопросом.
- Показывает блок диагностики:
  - какие вкладки найдены,
  - сколько строк считано,
  - какие колонки найдены/не найдены,
  - сколько задач попало в итоговый реестр.

## Структура проекта

- `index.html` — разметка страницы
- `styles.css` — стили
- `app.js` — логика парсинга, аналитики, фильтров, экспорта
- `vendor/xlsx.full.min.js` — локальная библиотека SheetJS (добавляется вручную)
- `README.md`

## Как запустить

### Вариант 1: просто открыть файл

1. Убедитесь, что `vendor/xlsx.full.min.js` лежит по нужному пути (см. раздел ниже).
2. Откройте `index.html` в браузере.
3. Нажмите `Выбрать .xlsx файл` и загрузите ваш Excel.

### Вариант 2: через локальный сервер (рекомендуется для кнопки автозагрузки `plan.xlsx`)

Из папки проекта:

```bash
python3 -m http.server 8000
```

Откройте в браузере:

- [http://localhost:8000](http://localhost:8000)

После этого кнопка `Попробовать загрузить plan.xlsx рядом с приложением` работает стабильнее.

## Куда положить xlsx.full.min.js

1. Скачайте `xlsx.full.min.js` (SheetJS Community Edition).
2. Положите файл сюда:

- `vendor/xlsx.full.min.js`

Итоговый путь должен быть именно таким:

- `./vendor/xlsx.full.min.js`

Без этого файла парсинг `.xlsx` в браузере не заработает.

## Ожидаемые столбцы в Excel

Приложение ищет заголовки нестрого (без учета регистра, с нормализацией пробелов) и по подстрокам:

- Категория: `категория`
- Этап: `этап`
- Описание задачи: `задач`
- Комментарий: `комментар`
- Релиз 1.20: `релиз 1.20`
- Релиз 1.21: `релиз 1.21`
- Scope: `входит в скоуп`

Если часть колонок не найдена, приложение покажет предупреждение в `Диагностике` и обработает всё, что возможно.

## Бизнес-правила, которые реализованы

- Оценка (SP) игнорируется.
- Цвета ячеек игнорируются.
- `Входит в скоуп 01.07` нормализуется к `ДА | НЕТ | ?`.
- Релизные колонки могут содержать типы работ через `/`.
- Поддерживаемые типы:
  - `разработка`
  - `тестирование`
  - `аналитика`
  - `внедрение`
- Если обе релизные колонки пустые -> задача `без релиза`.
- Вкладка с названием, содержащим `общепроект`, считается командой `Общепроектные`.
- Неизвестные значения типов работ не ломают приложение, а пишутся в `console.warn`.

## Ограничения

- Файл должен быть в формате `.xlsx`.
- Для чтения `.xlsx` требуется локальный `vendor/xlsx.full.min.js`.
- Автозагрузка `plan.xlsx` зависит от режима запуска и ограничений браузера; upload всегда доступен.
- Нет серверной части и нет сохранения состояния между перезагрузками страницы.
